## 3、功能设计

### 3.1、鉴权

#### 3.1.1、功能描述

新增通商E链业务网银兑付权限配置功能需要用户具有相应的权限才能访问和操作。

**权限控制：**

- 列表查看权限：所有授权用户
- 新增权限：仅项目维护人员（通过 `isEdit` 控制）
- 删除权限：仅项目维护人员（通过 `isEdit` 控制）

**权限标识：**

- `isEdit`：控制新增和删除按钮的显示
- `projectDetailsIsAdmin`：项目详情页管理员权限标识

#### 3.1.4、业务处理流程

**1. 列表展示流程**

- 进入项目详情/项目参数调整申请详情页面
- 加载 TeamworkSettlerList 组件
- 调用 getListValue 接口获取子公司信息列表
- 展示子公司信息列表（客户类型、客户类型名称、客户名称）

**2. 新增流程**

- 点击新增按钮
- 打开新增弹窗
- 默认设置客户类型为"公司客户"（customertype='01'）
- 客户类型和客户类型名称字段为只读
- 点击客户名称输入框，打开客户名称选择弹窗
- 选择客户后，自动填充客户 ID 和客户名称
- 验证数据：
  - 客户名称不能为空
  - 同一客户 ID 不能重复添加
- 调用 addTeamWorkSettlerPageVo 接口提交新增请求
- 新增成功后刷新列表

**3. 删除流程**

- 选择要删除的子公司信息
- 点击删除按钮
- 弹出确认对话框
- 确认后调用 deleteTeamWorkSettlerPageVo 接口
- 删除成功后刷新列表

**4. 客户名称选择流程**

- 点击客户名称输入框
- 打开 Participantname 组件弹窗
- 设置参数：
  - customerType='01'（公司客户）
  - type='selectCustomerInfoPage'
  - source='SubentCortAcctList'
  - customerId=核心企业 ID
- 选择客户后，返回客户 ID 和客户名称
- 自动填充到表单中

#### 3.1.5、字段保护设计

**1. 项目类型字段保护**

| 字段名 | 字段说明 | 类型 | 是否必填 | 默认值 | 备注 |
| --- | --- | --- | --- | --- | --- |
| ProjectType | 项目类型 | String | 是 |  | 永远不允许修改，只读 |

**保护规则：**

- 前端：设置 `colreadonly: '1'`，禁止用户编辑
- 后端：返回数据时不包含可编辑标识
- 权限：所有用户均无修改权限

**实现方式：**

```javascript

  colreadonly: '1',  // 只读
  iscolrequired: '1',
  collimit: 50,
  datatype: 'S',
  coltype: 'input',
  colwidth: 200
}
```
