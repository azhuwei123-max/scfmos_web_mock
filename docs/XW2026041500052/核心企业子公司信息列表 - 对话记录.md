## 二、开发实施阶段

### 2.1 TeamworkSettlerList 组件修改

**修改内容：**

1. **新增弹窗表单配置**
   - 本地写死 formTempList，不再调用 getTemplateInfo 接口
   - 客户类型默认设置为"公司客户"（customertype='01'）
   - 客户类型和客户类型名称字段设置为只读

2. **移除接口调用**
   - 移除 getTemplateInfo 接口调用
   - 保留 getTemplateFunc 使用（用于列表模板）

3. **新增按钮功能**
   - 点击新增按钮打开弹窗
   - 弹窗中默认设置客户类型
   - 客户名称通过 Participantname 组件选择

4. **删除按钮功能**
   - 点击删除按钮弹出确认对话框
   - 确认后调用 deleteTeamWorkSettlerPageVo 接口
   - 删除成功后刷新列表

### 2.2 后端 API 接口新增

**新增接口：**

1. **新增核心企业子公司信息**
   - 接口地址：`/system/paramAdjust/AddTeamWorkSettlerPageVo`
   - 请求方式：POST
   - 入参：
     | 参数名 | 参数类型 | 必填 | 说明 |
     |--------|----------|------|------|
     | objectType | String | 是 | 对象类型（ProjectParamAdjust/TeamWorkProject） |
     | objectNo | String | 是 | 对象编号（serialno） |
     | customerID | String | 是 | 客户 ID |

2. **删除核心企业子公司信息**
   - 接口地址：`/system/paramAdjust/deleteTeamWorkSettlerPageVo`
   - 请求方式：POST
   - 入参：
     | 参数名 | 参数类型 | 必填 | 说明 |
     |--------|----------|------|------|
     | serialno | String | 是 | 子公司信息编号 |

### 2.3 detail.vue 组件更新

**修改内容：**

1. **导入 TeamworkSettlerList 组件**
